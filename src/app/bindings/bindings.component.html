There are three types of bindings in angular:

Data Binding Description
One-way data binding From Component to View template
One-way data binding From View template to Component
Two-way data binding Both from View to Component and from Component to View


Interpolation:

Looking at Interpolation, which is a one way binding that flows from component to the view:

String operations:

This text here: {{ interpolationExample }}, is being displayed through interpolation. It is not hard-coded into the html, but instead is a string declared in the typescript of this component.

Interpolation is coded using double flower braces, with the variable in them. We can also insert strings into the interpolation syntax.
For example: {{ "This text is interpolated: " + interpolationExample }}

Arithmetic:

But wait, theres more! You can even perform arithmetic operations within interpolation syntax. 100 + 200 + 300 = {{ 100 + 200 + 300 }}

Ternary Operator:

Conditional operators or ternary operators can also be employed in interpolation.
In the following example, we check if a variable exists and if it does, we bring it to the view via interpolation.

This variable exists: {{interpolationExample? interpolationExample : "No."}}
This variable does not exist: {{ noInterpolationExample? noInterpolationExample: "No variable :/"}}

PS: Hover over highlighted text for tips.

Images:

We can also display images, by specifying the image path via interpolation.

Functions:

Interpolation can also be used to display the return object of a function. For example, this function will concatenate two strings, which have been stored in variables.

First name: {{firstName}}, Last name: {{lastName}}

Full name: {{getFullName()}}

Property Binding:

Property binding also works the same way as Interpolation, transporting data from component class properties to view element properties
